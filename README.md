# TaskManager.API
Описание

REST API для управления задачами, разработанное в качестве тестового задания. Проект демонстрирует подходы к проектированию масштабируемых backend-приложений на платформе .NET 8 с использованием PostgreSQL.

Архитектурные решения и подходы

Для обеспечения расширяемости и простоты поддержки приложения был выбран подход, основанный на принципах Clean Architecture (слоистой архитектуры), адаптированный под структуру монолитного Web API:

Разделение на слои (по папкам):

Domain: Содержит основные бизнес-сущности (TaskItem, TaskType). Ни от чего не зависит.

Infrastructure: Слой доступа к данным. Содержит AppDbContext и конфигурации Entity Framework.

Application: Бизнес-логика, сервисы (TaskService) и DTO (Data Transfer Objects). Разделение моделей БД и API предотвращает утечку данных и упрощает версионирование API.

API: Контроллеры, Program.cs, настройка DI-контейнера.

Внедрение зависимостей (Dependency Injection):
Логика изолирована за интерфейсом ITaskService. Это позволяет легко мокать сервисы при написании Unit-тестов.

Code-First подход и Автоматические миграции:
База данных проектируется через код. Для удобства проверки тестового задания, применение миграций (db.Database.Migrate()) настроено автоматически при старте приложения (в Program.cs).

Предзаполнение данных (Data Seeding):
Типы задач предзаполняются через метод OnModelCreating встроенными средствами EF Core.

Инструкция по запуску

Требования

.NET 8 SDK

Docker и Docker Compose (для запуска PostgreSQL)

Шаги

Запустите базу данных PostgreSQL с помощью Docker Compose:

docker-compose up -d


Перейдите в папку проекта:

cd TaskManager.API


(Опционально) Создайте первичную миграцию, если её нет:

dotnet ef migrations add InitialCreate


Запустите приложение:

dotnet run


Перейдите в браузере по адресу http://localhost:5xxx/swagger для тестирования API через Swagger UI.
