# TaskManager.API

> REST API для управления задачами, разработанное в качестве тестового задания. Проект демонстрирует подходы к проектированию масштабируемых backend-приложений на платформе **.NET 8** с использованием **PostgreSQL**.

---

## Архитектурные решения и выбор инструментов

Для обеспечения расширяемости и простоты поддержки был выбран подход, основанный на принципах **Clean Architecture** (слоистой архитектуры), адаптированный под структуру Web API.

### Разделение на слои

| Слой | Описание |
|------|----------|
| **Domain** | Основные бизнес-сущности (`TaskItem`, `TaskType`). Ни от чего не зависит. |
| **Infrastructure** | Слой доступа к данным. Содержит `AppDbContext`, конфигурации EF Core и глобальный обработчик ошибок (`GlobalExceptionHandler`). |
| **Application** | Бизнес-логика, сервисы (`TaskService`), валидаторы и DTO (Data Transfer Objects). Разделение моделей БД и API предотвращает утечку данных и упрощает версионирование API. |
| **API** | Контроллеры, `Program.cs`, настройка DI-контейнера. |

### Ключевые решения и инструменты

- **Внедрение зависимостей (Dependency Injection):** логика изолирована за интерфейсом `ITaskService`.
- **Code-First и автоматические миграции:** база данных проектируется через код. Применение миграций (`context.Database.Migrate()`) происходит автоматически при старте.
- **Предзаполнение данных (Data Seeding):** справочник "Типы задач" предзаполняется через `HasData` в `OnModelCreating`.
- **Пагинация (Pagination):** получение списка задач возвращает пагинированный ответ `PagedResult`.
- **Отмена операций (CancellationToken):** проброшена на все уровни для предотвращения утечки ресурсов.
- **Serilog:** заменяет стандартный логгер, обеспечивает структурированное логирование.
- **FluentValidation:** правила валидации вынесены отдельно от моделей (например, `CreateTaskDtoValidator`).
- **RFC 7807 (ProblemDetails):** глобальный перехват ошибок с унифицированным форматом ответа через `IExceptionHandler`.
- **xUnit + Moq + EF Core InMemory:** реализованы изолированные юнит-тесты на слой бизнес-логики.

## Инструкция по запуску

Проект полностью контейнеризован. API и база данных настроены на работу в одной сети.

### Требования

- [Docker](https://www.docker.com/) и Docker Compose

### Шаги для запуска

1. Откройте терминал в корневой папке проекта (где находится `docker-compose.yml`).
2. Выполните команду для сборки и запуска:

   ```bash
   docker-compose up -d --build
   ```

3. Docker автоматически скачает образ PostgreSQL, соберет образ API, запустит контейнеры и накатит миграции.
4. Откройте браузер и перейдите по адресу:

   👉 **http://localhost:8080/swagger**

5. Вы можете тестировать все CRUD операции напрямую через интерактивный интерфейс Swagger.
