# TaskManager.API

> REST API для управления задачами, разработанное в качестве тестового задания. Проект демонстрирует подходы к проектированию масштабируемых backend-приложений на платформе **.NET 8** с использованием **PostgreSQL**.

---

## Архитектурные решения и подходы

Для обеспечения расширяемости и простоты поддержки был выбран подход, основанный на принципах **Clean Architecture** (слоистой архитектуры), адаптированный под структуру монолитного Web API.

### Разделение на слои

| Слой | Описание |
|------|----------|
| **Domain** | Основные бизнес-сущности (`TaskItem`, `TaskType`). Ни от чего не зависит. |
| **Infrastructure** | Слой доступа к данным. Содержит `AppDbContext` и конфигурации Entity Framework. |
| **Application** | Бизнес-логика, сервисы (`TaskService`) и DTO (Data Transfer Objects). Разделение моделей БД и API предотвращает утечку данных и упрощает версионирование API. |
| **API** | Контроллеры, `Program.cs`, настройка DI-контейнера. |

### Ключевые решения

- **Внедрение зависимостей (Dependency Injection):** логика изолирована за интерфейсом `ITaskService`, что позволяет легко мокать сервисы при написании Unit-тестов.
- **Code-First и автоматические миграции:** база данных проектируется через код. Применение миграций (`db.Database.Migrate()`) настроено автоматически при старте приложения в `Program.cs`.
- **Предзаполнение данных (Data Seeding):** типы задач предзаполняются через метод `OnModelCreating` встроенными средствами EF Core.

## Инструкция по запуску

### Требования

- [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
- [Docker](https://www.docker.com/) и Docker Compose (для запуска PostgreSQL)

### Шаги

1. **Запустите базу данных PostgreSQL** с помощью Docker Compose:

   ```bash
   docker-compose up -d
   ```

2. **Перейдите в папку проекта:**

   ```bash
   cd TaskManager.API
   ```

3. **(Опционально)** Создайте первичную миграцию, если её нет:

   ```bash
   dotnet ef migrations add InitialCreate
   ```

4. **Запустите приложение:**

   ```bash
   dotnet run
   ```

5. Перейдите в браузере по адресу `http://localhost:5xxx/swagger` для тестирования API через **Swagger UI**.
